*추가설명*
"Application layer의 주고받는 메시지를 직접 구현하라"는 말은, WebSocket이나 다른 프로토콜을 사용하더라도 애플리케이션 레벨에서 교환되는 메시지의 구조를 직접 정의하고, 이를 송수신하도록 코드를 작성하라는 의미입니다. 즉, 서버와 클라이언트가 주고받는 메시지 포맷과 처리 방식을 직접 설계해야 한다는 뜻입니다. 라고 하네요

일주일 정도밖에 안남아서 다대다 채팅보단 1대1 채팅으로 구현하면 더 편할 것 같습니다. 작동 방식은 다음과 같습니다.

1. 프로그램 시작 시 닉네임 설정: 클라이언트를 실행할 때, 닉네임이나 이름을 입력받아 채팅 상대방에게 표시될 이름으로 사용.

2. 서버-클라이언트 연결: 서버와 클라이언트 간의 WebSocket을 통해 1대1로 연결.
각 메시지에 누가 보낸 것인지 알 수 있도록 닉네임을 함께 전송.

3. 메시지 전송 및 수신: 로그인 없이 연결된 상태에서 바로 메시지를 주고받으며, 상대방의 메시지가 수신되면 화면에 표시.
